  - Changed `do_nothing` penalty from `-0.18` to `-0.20`.

    - and the last 2 diagnoses both pushed `ent_coef = increase_small`,

[2026-04-11] Iteration 3 - Earlier History Branch Activation
- Modified `src/core/llm_client.py`
  - Added summary-level override:
    - If `success_rate <= 0.25`, `avg_return <= 3.0`, and `entropy_trend in {falling, flat}`,
    - then a passive `mixed_or_uncertain` diagnosis is upgraded to `exploration_collapse`,
    - with adjustment `ent_coef = increase_small`.
  - Relaxed history-aware trigger from needing 2 prior records to needing 1 prior record.
  - New history-aware trigger conditions:
    - previous round was not `accepted_improved`,
    - previous diagnosis already used `ent_coef = increase_small`,
    - current diagnosis again wants `ent_coef = increase_small`,
    - previous and current entropy trends both remain `falling` or `flat`,
    - current success rate still `<= 0.25`.
  - When the trigger fires, `history_aware` now switches to:
    - `n_steps = increase_small`
    - `gamma = decrease_small`
    - keep the other 3 parameters unchanged.
- Reason for the change:
  - In Iteration 2 smoke tests, both `single_shot` and `history_aware` stayed at `mixed_or_uncertain` and produced no parameter updates, so no group difference could emerge.

[2026-04-11] Iteration 10 - History-Aware Plateau Escape
- Modified `src/core/llm_client.py`
  - Added a second history-aware correction branch for low-success plateaus.
  - Trigger conditions:
    - at least 2 prior history records exist,
    - previous 2 rounds and the current round all have `success_rate = 0.0`,
    - previous 2 outcomes were `accepted_improved`,
    - current `avg_return < 1.0`,
    - previous diagnosis and current diagnosis both indicate `training_instability`.
  - When triggered, `history_aware` switches away from repeated stabilization and proposes:
    - `learning_rate = increase_small`
    - `gamma = increase_small`
    - keep the other 3 parameters unchanged.
- Reason for the change:
  - In Iteration 9, collapse was reduced, but both groups could still converge onto the same conservative `training_instability` adjustment path.
  - This branch makes `history_aware` explicitly use historical evidence to escape a low-success plateau that `single_shot` cannot identify from the current round alone.

[2026-04-11] Freeze Decision - Iteration 6 Style Environment
- Decision:
  - Freeze the environment around the Iteration 6 reward profile as the transition candidate for later CybORG migration.
- Why this version was selected:
  - Iteration 4/5 kept a clear `single_shot` vs `history_aware` difference, but still had too much `deploy_decoy` dominance.
  - Iteration 8/9 reduced collapse more aggressively, but the group difference weakened or disappeared.
  - Iteration 6 was the best compromise observed so far:
    - it reduced the previous trivial `patch` collapse,
    - preserved a clear and interpretable `history_aware` branch,
    - and still produced a positive performance gap for `history_aware`.
- Practical freeze choice:
  - Reverted the environment-side reward profile from the later Iteration 7/8 adjustments back toward the Iteration 6 values.
  - Kept later infrastructure fixes that are clearly beneficial:
    - parameter-space expansion with `n_steps` and `gamma`,
    - batch-size cleanup,
    - guardrail-safe fallback diagnoses.
